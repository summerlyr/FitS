# Security Policy

## Reporting a vulnerability

Please do not open a public issue for a suspected security vulnerability.
Use GitHub's **Report a vulnerability** option in the repository's Security tab
to submit a private report with reproduction steps and the affected version.

You should receive an initial response within seven days. After a fix is
available, the issue and any credit can be disclosed by mutual agreement.

## Scope

FitS is currently a local-first prototype with no account system or backend.
Reports involving unintended disclosure, unsafe file handling, or persistent
data corruption are in scope. General fitness or medical advice concerns should
be filed as regular issues rather than security reports.
